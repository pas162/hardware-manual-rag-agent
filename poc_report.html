MCP Server Shipped
Model Context Protocol is an open standard for tool-calling. The server is ~100 lines of Python (FastMCP) and exposes three tools the agent can call at any point during a chat.
search_um
Semantic search over all manual content. Returns ranked chunks with citations.
register_lookup
Exact SQL lookup for register addresses, reset values, and bit fields.
get_figure
Fetches a diagram by ID and returns the actual image for the LLM to see.
Database Both live
Two stores, each the right tool for its job: ChromaDB finds passages by meaning; SQLite returns exact register values that must never be guessed.

Register Schema
Register tables (those with "Bit", "Symbol", "R/W", "Reset" columns) are parsed directly into SQLite — without going through the embedding model. This gives the agent a deterministic SQL lookup path that never guesses a value.

Note on dual storage
Register data ends up in both stores:
• SQLite — exact structured rows, used by register_lookup for precise queries
• ChromaDB — chunker reads SQLite and creates register_row chunks that are embedded, enabling the register content to also be found via semantic search

Embedding & Indexing
Each chunk is converted to a vector by a local embedding model and stored in ChromaDB. This enables meaning-based search — finding the right passage even when the user phrases the question differently.
